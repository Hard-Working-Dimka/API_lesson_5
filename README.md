# Космический Telegram

Проект помогает скачивать фотографии SpaceX, NASA APOD, NASA EPIC, а так же публиковать их в телеграм канал.

Проект содержит следующие скрипты:

  * [Скрипт `auto_publisher_tg_bot.py`](#скрипт-auto_publisher_tg_botpy)
  * [Скрипт `fetch_nasa_apod.py`](#скрипт-fetch_nasa_apodpy)
  * [Скрипт `fetch_nasa_epic.py`](#скрипт-fetch_nasa_epicpy)
  * [Скрипт `fetch_spacex_images.py`](#скрипт-fetch_spacex_imagespy)
  * [Скрипт `image_file_tools.py`](#скрипт-image_file_toolspy)
  * [Скрипт `manual_publisher_tg_bot.py`](#скрипт-manual_publisher_tg_botpy)

### Как установить

Python3 должен быть уже установлен. 
Затем используйте `pip` (или `pip3`, есть есть конфликт с Python2) для установки зависимостей:
```
pip install -r requirements.txt
```
Далее добавьте в папку с проектом файл `.env` и скопируйте туда следующий код:
```
NASA_API_KEY=''
TG_API_KEY=''
PUBLICATION_PERIOD=''
CHAT_ID=''
PATH=''
IMAGE_QUANTITY=''
DATE=''
LAUNCH_ID=''
```
NASA_API_KEY - можно получить на сайте [NASA](https://api.nasa.gov/).

TG_API_KEY - получите в Telegram у [Bot_Father](https://telegram.me/BotFather).

PUBLICATION_PERIOD - период публикации каждой картинки в автоматическом режиме. Можно использовать сокращенные формы записи, например, `1h` - 1 час,`1s` - 1 секунда, `1m` - 1 минута, `1d` - 1 день, `1wk` - 1 неделя. Более подробное описание можно прочитать в [документации](https://pypi.org/project/pytimeparse/).

CHAT_ID - ссылка на Telegram канал через `@` (`t.me` не включать!).

PATH - путь до каталога с файлом/файлами.

IMAGE_QUANTITY - количество фотографий для загрузки.

DATE - дата, когда был сделан снимок. Формат даты `ГГГГ-ММ-ДД`.

LAUNCH_ID - id запуска SpaceX.

**Внимание!** Не все переменные обязательные, зависит от конкретного запуска скрипта. Описание запуска каждого скрипта смотри ниже.

## Скрипт `auto_publisher_tg_bot.py`

Скрипт предназначен для отправки фотографий в Telegram канал в автоматическом режиме. Автоматический режим предусматривает бесконечную отправку фотографий из каталога с задержкой. Если фотографии в каталоге закончатся, то скрипт их отправляет по новой, только в случайном порядке.

Запуск скрипта осуществляется через консоль. 
```
python auto_publisher_tg_bot.py
```
Данная команда поддерживает опциональные аргументы:

`-p` путь каталога с фотографиями. Значение по умолчанию - каталог `images`, который располагается в каталоге с проектом. Ввод пути осуществлять внутри кавычек `"path"`

`-per` период отправки фотографий. Значение по умолчанию - 4 часа. Это время, через которое осуществляется отправка каждой фотографии, кроме первой, которая отправляется моментально после запуска скрипта. 

Также вместо аргументов командной строки можно пользоваться переменными окружения: `PATH` и `PUBLICATION_PERIOD`.

**Обязательными переменными окружения являются:**
* TG_API_KEY
* CHAT_ID

## Скрипт `fetch_nasa_apod.py`

Скрипт используется для загрузки фотографий NASA APOD (Astronomy Picture of the Day).

Запуск скрипта осуществляется через консоль. 
```
python fetch_nasa_apod.py
```
Данная команда поддерживает опциональные аргументы:

`-p` путь к каталогу, куда будут загружаться фотографии. Значение по умолчанию - каталог `images`, который располагается в каталоге с проектом. Ввод пути осуществлять внутри кавычек `"path"`

`-q` количество загружаемых фотографий. Значение по умолчанию - 1. 

Также вместо аргументов командной строки можно пользоваться переменными окружения: `PATH` и `IMAGE_QUANTITY`.

**Обязательной переменной окружения является:**
* NASA_API_KEY

## Скрипт `fetch_nasa_epic.py`

Скрипт используется для загрузки фотографий NASA EPIC (Earth Polychromatic Imaging Camera). Скрип загружает все фотографии сделанные в указанный день.

Запуск скрипта осуществляется через консоль. 
```
python fetch_nasa_epic.py
```
Данная команда поддерживает опциональные аргументы:

`-p` путь к каталогу, куда будут загружаться фотографии. Значение по умолчанию - каталог `images`, который располагается в каталоге с проектом. Ввод пути осуществлять внутри кавычек `"path"`

`-d` дата, когда был сделан снимок. Значение по умолчанию - сегодня. 

Также вместо аргументов командной строки можно пользоваться переменными окружения: `PATH` и `DATE`.

**Обязательной переменной окружения является:**
* NASA_API_KEY

Данная команда поддерживает опциональные аргументы:

`-p` путь к каталогу, куда будут загружаться фотографии. Значение по умолчанию - каталог `images`, который располагается в каталоге с проектом. Ввод пути осуществлять внутри кавычек `"path"`

`-id` id запуска. Значение по умолчанию - последний. 

Также вместо аргументов командной строки можно пользоваться переменными окружения: `PATH` и `LAUNCH_ID`.

**Обязательных переменных окружения нет!**

## Скрипт `image_file_tools.py`

Вспомогательный скрипт. Содержит функции:

`get_file_extension` - используется для получения расширения файла из url. Обязательным аргументом является url. Функция возвращается расширение файла в формате `.extension`

`download_image` - используется для загрузки фотографий. Обязательные аргументы url фотографии, а так же путь, куда необходимо ее установить.

## Скрипт `manual_publisher_tg_bot.py`

Скрипт предназначен для отправки фотографий в Telegram канал в ручном режиме. Если в пути не указана фотография, то скрипт выберет случайную из указанного каталога.

Запуск скрипта осуществляется через консоль. 
```
python manual_publisher_tg_bot.py path
```
Здесь `path` - путь каталога с фотографиями или путь фотографии. Его необходимо заменить. Ввод пути осуществлять внутри кавычек `"path"`

**Обязательными переменными окружения являются:**
* TG_API_KEY
* CHAT_ID
